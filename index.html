Josh Bloom

We each have our own strengths, weakness, and goals when it comes to establishing paths within the field of Web Design & Development. For me I began my career in design and it is why I have such a focus on the smallest of details. That same level of care is evident in my development, every element is carefully considered and chosen based on maintaining careful semantics and optimization.

An artist creates for themselves, communicating and finding inspiration from within. Designers however communicate to others with specific goals. Inspiration can be found in so many locations. This is why I have gathered a library of components to help fuel and structure future projects.

Component Library

Header

Establishing the company brand is done through the placement of the logo within the top-left. Users are then able to access site content through a main navigation to the right, returning to the homepage by clicking on the logo itself. Existing users are also able to login via the right-most navigation item.

How might I build this?

This component (header, section, or div) contains both the logo (img, h1) and the navigation (nav) with nested links (ul, li, a). In order for the logo to function as an escape hatch, it might also need to be linked (a). The last navigation link, perhaps an account login (form, alternately styled a).

CTA

Just below the header, this Call-to-Action is used to capture the user's attention, focusing it on a specific action. In this case, conducting a search.

How might I build this?

This component (section, div), begins with a large heading (h1-h2) followed by a sub-heading tagline(h2-h5, p). Underneath is a search form (form). The form contains a text input (input), and a submit button (input, button).

How It Works

In order to convince users to download the advertised application it's important to demonstrate all the primary benefits of doing so. Here we are provided with three distinct features that showcase the inner workers of the web app: Realtime Stats, Set Goals, and Track Progress.

How might I build this?

This component (section, div), begins with a heading (h1, h2). Beneath this users are presented with information about 3 main features (articles, sections, list items, div). Inside of each feature element users are provided with a title (h2, h3), and a description (p).

* The images I'm not sure about... they would either be images tags (img) before the title or background-images applied to the appropriate feature.

Features

Convincing users to buy into any product or service requires the ability to sell them on the many benefits of doing so. This layout accomplishes this task by providing a grid of six distinct features, each establishing a specific impact that the product/service would have on their day-to-day lives.

How might I build this?

This component (section, div), begins with a heading (h1, h2). Beneath this users are presented with information about 6 features (articles, sections, list items). Inside of each feature element users are provided with a title (h2, h3), and a description (p).

* The images I'm not sure about... they would either be images tags (img) before the title or background-images applied to the appropriate feature.
